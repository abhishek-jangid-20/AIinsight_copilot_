11 — API Gateway Routes

server/src/routes/repositories.js · server/src/models.js

🌐 Node.js API Gateway — Route Map

All routes require requireAuth (JWT middleware)

📂 Repository CRUD

  • GET / — list user repos
  • POST /github — clone GitHub URL
  • POST /zip — upload ZIP (multer)
  • GET /:id — repo metadata
  • GET /:id/files — parsed file list
  • DELETE /:id — cascade delete (Chroma + Chats + MongoDB)

🔍 Semantic Search

  • POST /:id/search
  • Proxies → Embedding Service
  • Accepts: { query, limit 1–20 }
  • Returns: top matching chunks
  • Auth: only user's own repos

💬 RAG Chat

  • POST /:id/chat
  • Find/create Chat document
  • Append user message
  • Proxies SSE → RAG Service
  • Stream tokens to client
  • Save assistant reply to DB
  • GET /:id/chats — history

📄 Documentation

  • POST /:id/docs
  • Proxies → Documentation Service
  • Returns: readme, architecture, setup, modules
  • Requires: analysis ready

🕸️ Graph & Explain

  • GET /:id/graph → Analysis Service
  • Dependency graph nodes+edges
  • POST /:id/explain → Analysis Service
  • File purpose + symbol list

🔒 Security

  • requireAuth middleware on all routes
  • loadOwnedRepository() — IDOR protection
  • Zod schema validation
  • Multer: 150MB file size limit
  • GitHub URL regex validation

📊 MongoDB Models

  • User — email, name, passwordHash
  • Repository — status, analysis, sourceType
  • Chat — messages[], repositoryId
  • ChatMessage — role, content, citations

📡 SSE Streaming

  • content-type: text/event-stream
  • cache-control: no-cache
  • x-chat-id: header returned
  • TextDecoder reads RAG chunks
  • extractMessagePayload() helper
